核心原理：現代廚房運作模式

本系統的技術哲學在於將「醫療人工智慧應用」類比為一間高效能的現代廚房 (Modern Kitchen)。在這種架構下，醫療需求不再是被單一處理，職責經過精密的流水線作業產出結果。

1. 外場接收層 (Interaction Layer)

如同餐廳的領檯與服務生（包含圖中左側的 Telegram/LINE/WhatsApp Bot），負責接收顧客（醫護或病患）的訂單，並將其轉換為內場可理解的 Webhook 標準格式。

2. 內場經理層 (Kitchen Manager)

這是系統的中樞 Webhook 管理器與 API Gateway。它負責「排單」與「備料」，判斷任務類型，並將其組合成合適的「LLM Prompt」傳遞給核心處理層。

3. 行政主廚與食譜引擎 (Master Chef & Recipe Engine)

核心大型語言模型（GPT-4/Claude）扮演主廚，依據 GitHub 中的「食譜庫（Codebase 與訓練手冊）」對數據進行處理，確保每一道「醫療建議」都符合最新的專業指引。

4. 硬體與基礎設施 (Infrastructure)

由 Railway 與 Zeabur 提供運算引擎（爐灶）與資料庫儲存（儲藏室），透過 CI/CD 流程確保系統能即時更新並 24 小時穩定運作。
